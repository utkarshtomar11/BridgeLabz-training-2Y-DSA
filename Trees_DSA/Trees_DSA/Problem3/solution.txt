e) Height comparison - Balanced vs Skewed BST:
----------------------------------------------------------------

   Our Balanced BST height:
   Longest path: 50 -> 30 -> 20 -> 10  (3 edges)
   --> Height = 3

   Completely Skewed BST (9 elements, inserted in sorted order):
   10 -> 20 -> 25 -> 30 -> 40 -> 50 -> 60 -> 70 -> 80
   (each node only has a right child, like a linked list)
   --> Height = 8  (n - 1 for n nodes)

   Conclusion:
   Balanced BST Height = 3  (efficient: O(log n) operations)
   Skewed BST Height   = 8  (inefficient: O(n) operations)
   Balanced BST is nearly 3x more efficient in this case!
